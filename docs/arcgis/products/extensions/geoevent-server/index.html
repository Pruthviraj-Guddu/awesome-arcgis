Introduction

ArcGIS GeoEvent Server can track dynamic assets that are constantly changing location such as vehicles, aircraft, and vessels or stationary assets such as weather and environmental monitoring sensors. In addition, it provides real-time situational awareness for coordinated field activities.

Code

  • Proxy that handles tokens for versions of ArcGIS GeoEvent Processor for Server and ArcGIS GeoEvent Extension for Server
  • The Cluster Simulator functions as a source of data feeding GeoEvents into a cluster of ArcGIS Servers running GeoEvent Extension.
  • Custom processors, adapters and transports for geoevent server.
  • ArcGIS GeoEvent Extension Samples:
    • MongoDB Outbound Connector for storing GeoEvents.
    • Hadoop Outbound Connector for storing GeoEvents in HDFS.
    • Twitter connectors for sending and receiving tweets
    • Instagram Inbound Connector for receiving Instagram feeds.
    • Trimble Connector for receiving raw data in the TAIP format.
    • RabbitMQ connectors for connecting to RabbitMQ Message Servers
    • ActiveMQ connectors for connecting to ActiveMQ Message Servers
    • KML Connector
    • Connector for receiving raw data in the NMEA format.
